# 材料送審管制總表轉換 — 計畫說明

## 目的

將詳細價目表（Excel）作為母本，填入材料送審管制總表（表5.1）Word 模板，產出完整多頁 Word 文件。

## 輸入

| 檔案 | 說明 |
|------|------|
| `data/02_成德-詳細價目表.xlsx` | 詳細價目表母本（Table 1 工作表） |
| `tables/table5.1/表5.1.docx` | 表5.1 模板 |

## 處理規則

### 資料篩選
- 起點：項次 `壹.三.1` 之後
- 項次須以 `壹.` 開頭
- 排除單位：`式`、`工`（可自訂 `--exclude-units`）
- 排除名稱含 `小計`、`合計`、`總價` 之項目
- 文字清洗：移除換行符、多餘空格、標點前空格（`clean_text()`）
- ⑤ → 窗（修正 PDF 轉檔錯誤）

### 分頁
- 動態分頁，依資料列實際高度決定每頁可容納組數
- 每頁可用高度：`DATA_AVAIL_TWIP`（11170）− `HEADER_H_TWIP`（2600）≈ 8570 twip
- 實際經 Word 驗證：181 項 → 15 頁
- 每頁之間插入分頁符號

### 表頭（每頁頂部 3 行）
1. `表5-1 材料設備送審管制總表-N`（14pt、粗體、置中、固定行高 24pt）
2. `工程名稱：...`（12pt、靠左）
3. `(監造單位使用) 第N頁共Σ頁 表單編號：E51-N`（12pt、最小行高 12pt）

### 儲存格格式
- 黑實線框（1pt、黑色）
- 段落行距：`exact` 240 twip
- 列高規則：`atLeast`（可微量擴展防裁切）
- 儲存格邊距（tcMar）：全部設 0
- 垂直置中

### 資料填入
- C7（協力廠商資料）、C8（型錄）：`V`
- C4（預定送審日期）：`施作前14日`
- C5（是否驗廠）：`否`
- C2（qty_unit）：數量 + 單位

### 標題列合併（R1 + R2）
| 合併 | 不併 |
|------|------|
| C0（項次）、C2（契約）、C3（是否取樣試驗）、C6（預定試驗單位）、C7~C12（送審資料）、C14（備註） | C1（材料名稱）、C4（預/實送審日）、C5（驗廠）、C13（審查結果） |

### 資料列合併（奇數列 + 偶數列）
| 合併 | 不併 |
|------|------|
| C0（流水號）、C2（數量+單位）、C3、C6、C7~C12、C14 | C1（奇=項次、偶=名稱）、C4、C5、C13 |

## v2.0 重大變更

| 項目 | 舊版 | v2.0 |
|------|------|------|
| 資料列產生 | `deepcopy` 範本列 | `etree.SubElement` 完全重建 |
| 列高規則 | `atLeast`（無段落約束） | `atLeast` + 段落 `exact` |
| 儲存格邊距 | 繼承範本 | `tcMar=0` |
| 表格寬度 | 繼承範本 | 明確設定 `5289 pct`（匹配範本） |
| 文字清洗 | 無 | `clean_text()` 移除斷行/空格 |
| 頁碼表頭 | 無 | 三行表頭（標題、工程名稱、頁碼） |

## 輸出

| 產出 | 說明 |
|------|------|
| `output/表5.1_完成.docx` | 完整多頁材料送審管制總表 |

## 使用方式

```bash
python -X utf8 tables/table5.1/convert_5.1.py --exclude-units 式 工
```
