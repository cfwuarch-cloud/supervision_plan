## 三、各腳本功能定義

### 3.1 前言

- **腳本**：`tables/foreword/convert_前言.py`
- **輸入**：文字模板（*待提供*）、專案設定檔
- **輸出**：`output/前言.docx`
- **內容**：公共工程委員會監造計畫製作綱要之前言部分 + 各級距適用說明

### 3.2 Ch1 監造範圍

- **腳本**：`tables/ch1/convert_1.py`
- **輸入**：文字模板（*待提供*）、`data/02_成德-詳細價目表.xlsx`
- **輸出**：`output/Ch1_監造範圍.docx`
- **價目表參數**：`--level N`（控制顯示階層，例如 `--level 2` 只列出小項）
- **內容**：
  - 依據（法規列表）
  - 工程概要（專案設定檔）
  - 主要施工項目及數量（從價目表讀取）
  - 適用對象
  - 名詞定義

### 3.3 Ch2 監造組織及權責分工

- **腳本**：`tables/ch2/convert_2.py`
- **輸入**：文字模板（*待提供*）、SVG 組織圖
- **輸出**：`output/Ch2_監造組織及權責分工.docx`
- **SVG**：用 python-docx 插入 SVG（或轉 EMF），組織架構圖
- **內容**：
  - 監造組織架構圖
  - 人員配置說明
  - 工作職掌（以職稱說明）
  - 權責分工表
  - 開工前協調會議

### 3.4 Ch3 品質計畫審查作業程序

- **腳本**：`tables/ch3/convert_3.py`
- **輸入**：文字模板（*待提供*）、流程圖 SVG、表3.1 docx 範本
- **輸出**：`output/Ch3_品質計畫審查作業程序.docx`
- **內容**：
  - 審查核定流程（含流程圖 SVG）
  - 審查時限
  - 不符合處理
  - 送審管制
  - 品管人員審查核定（級距三）
  - 表3.1 審查意見表

### 3.5 Ch4 施工計畫審查作業程序

- **輸入**：文字模板（*待提供*）、流程圖 SVG、表4.1/4.2 docx 範本
- **輸出**：`output/Ch4_施工計畫審查作業程序.docx`
- **內容**：

## 四、共通設計原則

### 4.1 文字內容管理

- 文字內容用 `data/chX_內容.md` 存放（由使用者另提供），腳本讀入後填入 docx 模板
- placeholder 格式：`{{專案名稱}}`、`{{主辦機關}}` 等，從專案設定檔統一填入

### 4.2 價目表讀取（Ch1）

- 沿用現有 `data/02_成德-詳細價目表.xlsx` 讀取邏輯
- 新增 `--level` 參數：
  - `--level 1`：大項（壹、貳…）
  - `--level 2`：小項（壹.一、壹.二…）— 預設
  - `--level 3`：細項（壹.一.1、壹.一.2…）
- 沿用既有篩選規則：排除單位 `式` `工`、排除 `小計` `合計` `總價`

### 4.3 SVG 流程圖

- 用 python 從零生成 SVG XML
- 遵循 AGENTS.md 規範：框內最後一行文字距框底部 **14px**
- 插入 docx 方式：轉為 EMF（python-docx 支援）或直接嵌入 VML
- 共用 SVG 工具函式可抽至 `common/svg_utils.py`

### 4.4 表格格式

- 沿用 `common/docx_table.py`（黑實線框、tcMar=0、vAlign=center）
- 行高規則：`atLeast` trHeight + `exact` 段落行高（LINE_H_TWIP=240）
- 文字清洗：沿用 `clean_text()` 移除斷行、多餘空格

## 五、執行優先順序

| 優先 | 章節 | 理由 |
|:----:|------|------|
| 1 | **Ch1 監造範圍** | 最簡單、先確立價目表階層參數 |
| 2 | **Ch2 監造組織** | SVG組織圖作為SVG流程圖前導 |
| 3 | **Ch3 品質計畫審查** | 流程圖＋表3.1，核心章節 |
| 4 | **Ch4 施工計畫審查** | 同Ch3模式，可快速複製 |
| 5 | **Ch6 設備功能運轉** | 機電條件章節 |
| 6 | **Ch8 品質稽核** | 級距三專用，文字為主 |
| 7 | **Ch9 文件紀錄管理** | 純文字章節 |
| 8 | **前言** | 最後再做以統整各章內容 |
| 9 | **merge_docx.py** | 最後再做 |

## 六、注意事項

1. **文字內容**待使用者提供 `data/chX_內容.md` 後，再整合進腳本
2. 各腳本完成後需更新 `AGENTS.md` 的常用指令區
3. 輸出檔建議加上 gitignore 規則避免誤傳（`output/` 已 gitignore）
4. 各章模板 docx 先以「空白範本」建立，待文字內容到位後再填入
